What Makes It Different

It answers "why is my Mac slow?" instead of just charting load.

Opinionated triage

The top of the menu collapses health, likely cause, next action, and alert thresholds into one readable operator block.

Flight Recorder

A rolling local history compares now vs 1m, 5m, and 15m ago so the plugin can explain what changed before the slowdown.

Safe action boundaries

Routine actions stay one click away. Disruptive actions are grouped and labeled so contributors do not bury risk in normal menu flows.

Dedicated install folder

A checkout-friendly swiftbar/ folder keeps SwiftBar away from dev files like node_modules while self-update still targets the repo.

Install

Two supported paths. Same updater.

Repository checkout

  1. Clone the repository into a stable folder.
  2. Point SwiftBar at the repo's swiftbar/ subfolder.
  3. If needed, run chmod +x system-monitor.5s.sh once.
  4. Use the in-menu updater for future releases.

Single plugin file

  1. Download `system-monitor.5s.sh` from the latest release.
  2. Drop it into an existing SwiftBar plugin folder.
  3. Mark it executable.
  4. The same updater replaces the local file in place.
Quick start 
brew install swiftbar
chmod +x system-monitor.5s.sh
open -g "swiftbar://refreshplugin?plugin=system-monitor.5s.sh"
